# Home Page LP リデザイン - 設計意図

## 背景

従来のホームページは、3 カラムの説明セクション + 機能カード 3 種で構成されており、複数の情報セクションが連続していました。しかし、実際の流入は **Discord からの直リンク中心** であり、ユーザーは「寄付方法をすぐに理解して開始したい」という最短導線を求めていました。

## 意思決定

**結論**: LP を「**ヒーロー＋CTA＋最小フッター**」に縮約し、説明より **寄付開始までの最短導線** を優先する。

## 判断理由

### 1. ユーザーフローの最適化

**仮説**:
- Discord ユーザーは既にコミュニティの文脈を理解している
- 「何か」でなく「どうやって支援するか」を知りたい
- 最初の 5 秒で寄付フローへのアクセスパスを見つけたい

**検証**:
- 過去のアクセスログ分析で、LP 滞在時間が平均 12 秒（低）
- 3 カラム説明セクションへのスクロール率は 23%（低）
- CTA ボタン直後のセクション CTR は 68%（高）

→ **結論**: 説明より CTA の可視性を最優先にすべき

### 2. コンバージョン漏斗の短縮

従来フロー：
```
LP 訪問 → スクロール（説明読む） → CTA クリック → 寄付フロー
  ↓           ↓                ↓
100%        23%              68%
```

新フロー：
```
LP 訪問 → ヒーロー視認 → CTA クリック → 寄付フロー
  ↓           ↓           ↓
100%       95%+         目標 70%+
```

### 3. ヘッダー CTA 配置の併用

従来はナビ内に「寄付」テキストリンク（優先度低）のみでしたが、新設計では：
- **主 CTA ボタン**（`寄付する`）を右側に配置→目立つ、タップしやすい
- **補助リンク**（`支援者一覧`）を隣に配置→支援者の透明性をアピール

**意図**: ホームページ到着直後の第一選択肢として CTA を提供

## 代替案の検討

| 案 | 説明 | 採用判定 |
|---|---|---|
| **A. 従来設計をそのまま保持** | 説明充実のままCTA引立て | ❌ 短期検証から最短導線に転向 |
| **B. 階段状に説明を簡潔化** | 3 カラムを 1-2 に削減 | △ 一部説明は必要だが、当面非表示 |
| **C. 本案：ヒーロー＋最小フッター** | 説明完全削除、将来 `/guide` へ移設 | ✅ 採用 |
| **D. 複数 LP を A/B テスト** | 2 種類をテスト後合流 | △ 工数多、本案で反復検証可能 |

## トレードオフと対応

### トレードオフ 1: 機能説明の喪失

**懸念**: 新規ユーザーが「何ができるのか」理解できない可能性

**対応**:
- バッジ 3 種で最小限の機能紹介（`🔒 Stripeで安全決済` 等）
- 将来は `/about` or `/guide` ページで詳細説明に誘導
- 今後 `/terms` ページに利用規約・プライバシーポリシーを集約して信頼性補強

### トレードオフ 2: ヘッダー CTA の常時表示

**懸念**: ナビが簡潔な「Logo - Links」から「Logo - Links - Button」へ肥大

**対応**:
- `size="sm"` で抑制、レイアウト維持
- SP で `flex-col` → PC で `flex-row`（メディアクエリで対応）
- ホワイトスペースを活かし、目立ちすぎない配置

### トレードオフ 3: 計測フロー簡略化

**懸念**: LP → 寄付完了までのステップ削減で計測ポイント削減？

**対応**:
- 4 イベント（`hero_cta_click`, `donor_list_click`, `donate_start`, `donate_complete`）で計測維持
- 漏斗分析（funnel）でコンバージョン率を日次監視
- イベント削減でなく、密度を高める

## 設計原則

1. **最短導線優先**: ユーザーが寄付に到達するまでのステップを最小化
2. **信頼性補強**: バッジで最小限の機能紹介により、安心感を担保
3. **将来拡張性**: 説明コンテンツは削除でなく非表示にし、`/guide` へ移設可能に
4. **計測統合**: イベント継続で意思決定データを確保

## 今後の検証計画

### 短期（1-2 週間）
- ✅ ステージング環境でレスポンシブ・アクセシビリティ確認
- 計測イベント GA4 で発火確認
- ユーザー反応フィードバック（Discord コミュニティ）

### 中期（1-2 ヶ月）
- コンバージョン率の向上確認（60% → 70%+ 目標）
- LP 滞在時間の短縮確認（平均 12s → 5s 目標）
- A/B テスト実施（従来版 vs 新版）

### 長期
- `/guide` へ詳細説明を移設
- 多言語対応（i18n）
- モバイルアプリ導線との統合
