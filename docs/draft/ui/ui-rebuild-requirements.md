## 共通要件
- UI はレスポンシブデザインを採用し、モバイル・タブレット・デスクトップで適切に表示されること
- ページは`home/`(root),`donate/`,`donors/`,`thanks/`,`terms/`の各パスでアクセス可能であること
- apiへのアクセス(oauth/やapi/)などは適切に実装する必要があるが、バックエンド実装や完全に機能するページは今回の実装において非対象とし、UI/UX要件に集中すること
- 団体名は"Minecraft Server 🍀"とする
- 全ページの表示言語は日本語とする

## `/home` 要件
- "みんなで創る、みんなの世界"をキャッチコピーとして表示し、寄付の重要性と影響を簡潔に説明する
- "寄付する"ボタンを設置し、クリックすると`/donate`ページへ遷移する
- LPとしての役割を果たすため、視覚的に魅力的でナビゲーションが直感的であること。最も創造性が高いデザインを採用すること

## `/donate` 要件
- UI フローは「注意書き表示 → Discord ログイン → 掲示同意チェックボックス (既定 OFF) → 単発¥300 / 月額¥300 / 年額¥3,000 の決済ボタン」の順で提示する
- 同意チェックやログインボタンは `GET /oauth/start` を呼んで `consent_public` を state Cookie に載せ、callback の TTL/署名エラークエリを解釈して UI で再試行を促す
- 表示名と同意状態は `/api/session` 経由で `sess` Cookie から取得し、TTL 超過を検知したら再ログインを要求する
- 寄付とは独立に掲示同意を ON/OFF できるトグルを用意し、`POST /api/consent` へ署名 Cookie 付きリクエストを送って即座に撤回/再同意できるようにする
- 決済ボタンは `POST /api/checkout/session` に許可されたモード/プランを送信し、Stripe Checkout URL が返ったら `success_url=/thanks` と `cancel_url=/donate` で遷移させる

## `/donors` 要件
- 公開リストでは Stripe 由来の表示名だけを並べ、金額・回数・順位は出さずにいつでも撤回できることを明記する
- `GET /api/donors` を `limit` (1〜100) と `order` (`desc`/`asc`/`random`) で呼び、既定は `order=desc`・`limit=100` としつつランダム表示を選べる UI を用意する
- ページ内に Discord ログイン＋`POST /api/consent` の導線を置き、Donors ページから直接同意撤回や再掲示ができるようにする
- 60 秒キャッシュの遅延とそれに伴う表示ラグを説明し、反映されない場合の待機/問い合わせ手順を提示する。
- `400/500` 応答時には「入力の見直し」「時間をおいて再試行」「サポート連絡」などの案内を表示し、API エラーコードと紐づける

## `/thanks`・周辺ページ要件
- Stripe Checkout 成功時に `/thanks` へ遷移させ、感謝メッセージ以外の CTA や特典案内を置かず、キャンセルは `/donate` に戻す
- 独自のサンクスメールは送信しない方針を UI でも明示し、通知は Stripe レシートのみであると伝える
- 寄付に対する感謝を伝えるための適切なデザインや表示が施されていること
- `/terms` ページ内のプライバシーセクションには簡潔なプライバシー方針と問い合わせ手段を記し、個別の問い合わせチャネルを案内する
